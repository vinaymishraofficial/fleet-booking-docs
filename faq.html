Frequently Asked Questions

Find answers to common questions about Fleet Booking System installation, configuration, and usage.

üåü General Questions

What is Fleet Booking System?

Fleet Booking System is a comprehensive fleet management application built for ERPNext. It enables organizations to manage vehicle bookings, track fleet usage, monitor drivers, and optimize routes with real-time GPS tracking capabilities.

What are the system requirements?

The system requires ERPNext version 13 or later, Python 3.8+, and MariaDB/MySQL. For GPS tracking features, you'll need a Google Maps API key. The system is compatible with Linux, macOS, and Windows environments.

Is Fleet Booking System free to use?

Yes! Fleet Booking System is open-source software released under the MIT License. You can use, modify, and distribute it freely. However, you may need to pay for third-party services like Google Maps API based on your usage.

Can I customize the system for my organization?

Absolutely! The system is built on ERPNext framework, making it highly customizable. You can modify workflows, add custom fields, create custom reports, and integrate with other systems. Check our Developer Guide for detailed instructions.

üöÄ Installation & Setup

How do I install Fleet Booking System?

Installation is straightforward with our automated installer. Run bench get-app fleet_booking followed by bench --site your-site install-app fleet_booking. For detailed steps, see our Installation Guide.

Do I need a Google Maps API key?

Yes, for GPS tracking and route optimization features, you need a Google Maps API key with the following APIs enabled: Maps JavaScript API, Geocoding API, and Directions API. You can get a key from the Google Cloud Console.

How do I configure the system after installation?

After installation, go to Fleet Booking Settings to configure Google Maps API, notification settings, and workflow parameters. You can also set up vehicle and driver masters in the ERPNext system.

Can I migrate from another fleet management system?

Yes, you can migrate data from other systems using ERPNext's data import tools. We recommend starting with vehicle and driver data, followed by historical booking records. Contact our support team for migration assistance.

‚ö° Features & Functionality

What types of vehicles can I manage?

The system supports all types of vehicles including cars, trucks, buses, motorcycles, and specialized vehicles. Each vehicle can have custom attributes like fuel type, capacity, maintenance schedules, and insurance details.

How does the GPS tracking work?

GPS tracking integrates with Google Maps to provide real-time vehicle location updates. Drivers can update their location through the driver dashboard, and fleet managers can monitor all vehicles on a centralized map interface.

Can employees book vehicles in advance?

Yes, employees can submit booking requests for future dates through the employee portal. The system checks availability, prevents conflicts, and routes requests through the configured approval workflow.

How are drivers assigned to bookings?

Drivers can be assigned automatically based on availability and proximity, or manually by fleet managers. The system considers driver schedules, vehicle assignments, and workload distribution for optimal assignments.

What reports are available?

The system includes comprehensive reports for vehicle utilization, driver performance, fuel consumption, maintenance schedules, booking analytics, and cost analysis. Custom reports can be created using ERPNext's report builder.

üîß Troubleshooting

GPS tracking is not working. What should I check?

First, verify your Google Maps API key is valid and has the required APIs enabled. Check browser permissions for location access, ensure stable internet connectivity, and verify the API key is correctly configured in Fleet Booking Settings.

Booking conflicts are not being detected. Why?

This usually happens due to timezone issues or incorrect time formats. Ensure your ERPNext system timezone matches your operational timezone, and verify that booking times are in the correct format (HH:MM:SS).

Email notifications are not being sent. How to fix?

Check your ERPNext email settings first. Ensure SMTP is configured correctly, then verify notification templates are active in Fleet Booking Settings. Also check the email queue for failed messages in ERPNext.

The driver dashboard is loading slowly. How to optimize?

Slow loading can be due to large datasets or network issues. Consider implementing data pagination, optimizing database queries, and enabling browser caching. For large fleets, consider archiving old booking records.

How do I backup my fleet data?

Use ERPNext's built-in backup functionality: bench --site your-site backup --with-files. We recommend daily automated backups and testing restore procedures regularly. Store backups in multiple locations for redundancy.

üîå API & Integration

Can I integrate with other systems?

Yes, the system provides comprehensive REST APIs for integration with HR systems, accounting software, mobile apps, and third-party fleet management tools. See our API Reference for detailed documentation.

Is there a mobile app available?

Currently, the system provides a responsive web interface optimized for mobile devices. Native iOS and Android applications are planned for future releases. The driver dashboard works excellently on mobile browsers.

How do I authenticate API requests?

The API supports multiple authentication methods including API keys, session-based authentication, and JWT tokens. For third-party integrations, we recommend using API keys with appropriate permissions.

Can I create custom API endpoints?

Yes, you can create custom API endpoints by adding functions to the api.py file with the @frappe.whitelist() decorator. See our Developer Guide for examples and best practices.

üìà Performance & Scaling

How many vehicles can the system handle?

The system can handle thousands of vehicles efficiently. Performance depends on your server specifications and database optimization. For large fleets (500+ vehicles), consider database indexing and caching strategies.

How to optimize system performance?

Enable Redis caching, optimize database queries, implement proper indexing, use CDN for static assets, and consider load balancing for high-traffic scenarios. Regular maintenance and monitoring are essential.

Can I run multiple instances?

Yes, you can deploy multiple instances for different regions or departments. Each instance can have its own database or share a common database with site-specific data isolation.
